## Key design decisions (pinned)

1. **"Allow all" sentinel for model/killzone sets**: `PostLLMParams` uses
   `frozenset()` semantics where an **empty frozenset = reject everything**
   (testable). "Allow all" is represented by a module-level sentinel
   `ALL = frozenset({"*"})` — rescore treats `{"*"}` as "no filter". This
   avoids the empty-set ambiguity. `default_post_params()` uses `ALL` for both
   (baseline = no filtering = today's behavior).

2. **R:R recompute in rescore**: reuses the EXACT formula from `validate_rr`
   (`entry_mid = (ez[0]+ez[1])/2; risk = abs(entry_mid - stop); rr = abs(target -
   entry_mid)/risk`) via a shared helper, so the rr_min filter is internally
   consistent with the trading layer. A degenerate (risk==0) taken trade is
   downgraded to no_trade (matches validate_rr's degenerate_stop branch).

3. **Objective = PerformanceReport.expectancy**: the single source of truth is
   `PerformanceReport._overall_metrics(traces)["expectancy"]`, which is
   `mean(actual_rr)` over all traces with no_trade/no_fill/expired = 0. I REUSE
   it (call `_overall_metrics`), never recompute the mean independently.
   `taken_trades` = filled count (win/loss/expired) on the re-scored set.

4. **MIN_TRADES guard**: `evaluate` returns `(-inf, metrics)` when
   `taken_trades < min_trades`. `metrics` always populated. `-inf` strictly
   below; finite at exactly `min_trades`.

5. **Walk-forward folds**: rolling, half-open `[start, start+train_days)` /
   `[start+train_days, start+train_days+test_days)`. **Per-fold disjointness
   asserted** (release blocker). step_days advances the window. Folds with
   empty train or test dropped. Timestamps parsed once to aware datetimes
   (naive → ET, matching report.py).

6. **Tie-break in search**: first-in-grid-order wins (deterministic). Documented.

7. **All-below-min_trades fallback**: returns the baseline (default) params with
   their score so the fold still records a comparable result. Documented.

8. **Aggregation in report**: reports BOTH mean-of-folds AND trade-weighted
   (pooled) for OOS expectancy/win-rate/PF/max_dd, so small-sample folds can't
   hide. Documented.

9. **Overfitting warning**: fires when (aggregate IS − aggregate OOS) > gap
   threshold (default 0.5R, documented) OR tuned OOS ≤ baseline OOS. The latter
   emits "Tuning added nothing; use baseline."

10. **Tier-2 recording**: cost = `n_configs × total_fires × $0.003` (one
    full-span backtest per config; folds do NOT multiply cost — a config's LLM
    output is fold-independent, recorded once and sliced per fold by
    timestamp). Per-config `runs/{config_hash}/` dirs. Cost gate mirrors Phase
    3A spend gate. Idempotent resume via BacktestEngine.
