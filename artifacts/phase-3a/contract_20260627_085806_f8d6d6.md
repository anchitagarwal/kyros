## Resolved design decisions (from blueprint open questions)

### D1 — CSV timezone
The real `workspace/data/nq_1min_data.csv` `date` column is **UTC, tz-naive**.
Verified: first row `2024-06-09 22:00:00` = Sunday 18:00 ET = CME futures open;
`2024-06-10 13:30:00` UTC = 09:30 ET (NY AM killzone open). DataLoader parses
`date` as UTC tz-aware. ReplayCandleSource converts to ET internally (already
does this via `tz_convert`).

### D2 — `end` bound semantics
**Inclusive** of `end` date (date-level). DataLoader filters
`timestamp >= start AND timestamp <= end + 1 day` (end is a date; we include
the full end day). Documented in code.

### D3 — `actual_rr` formula
- Win: `(exit_price - fill_price) / abs(fill_price - stop)` for long;
  `(fill_price - exit_price) / abs(fill_price - stop)` for short.
- Loss: negative of the same formula (exit_price = stop for a clean stop-out,
  giving exactly -1.0; a gap-through-stop gives worse than -1.0 — realized).
- `exit_price` for win = `alert.target`; for loss = `alert.stop`.
- expired: `actual_rr = None` (no resolution — spec says null for expired).
- no_fill / no_trade: `actual_rr = None`.
This matches the spec's "negative of the same formula" and the test's
"loss → actual_rr negative" / "win → actual_rr positive".

### D4 — Subsequent-candle sourcing in BacktestEngine
**Two-pass / full-frame approach**: BacktestEngine holds the full 1m parquet
frame separately. It drives `TradingLoop` via the `CandleSource` (no lookahead
to the loop). When an alert fires, it slices subsequent 1m candles **strictly
after** `alert.timestamp` from the full frame (the one allowed forward read,
isolated to OutcomeSimulator). The replay buffer holds ≥480 1m candles.

### D5 — Resume granularity
Keyed on `alert.timestamp` (ISO string). The spec says "skip candles whose
alert.timestamp is already in the file." Production TradingLoop emits at most
one alert per candle (cooldown suppresses same-killzone repeats), so
timestamp is a sufficient key. Composite key not needed.

### D6 — Golden match timeline source
PerformanceReport matches golden_alerts.json entries against **TradeTrace
traces** (the spec says "for each entry find if any trace within 15 min has
matching direction"). The blueprint raised "fires vs alerts" — the spec text
explicitly says "trace within 15 min", so we use traces. ±15 min symmetric
(forward or backward) — documented. Direction match: golden `direction` ∈
{long, short} vs trace `alert.bias`.

### D7 — Calibrator cooldown semantics
The calibrator mirrors production: it calls `cooldown.update()` on every fire
with a directional alert (bias derived from `htf_bias`), so `cooldown_active`
blocks reflect production tier behavior. This is required so calibrator
fire counts are comparable to BacktestEngine fire counts.

### D8 — Calibrator window warm-up
The calibrator evaluates the trigger on **every** candle the source emits
(no `is_warm` gate) — mirroring production `TradingLoop.run()`, which builds a
snapshot and evaluates on every candle. The test expectations
(`flat: total_fires=0`, `killzone_active: gate_blocks>0`, `sweep_and_fvg:
total_fires>0`) are met by choosing `n_bars` per scenario in the test fixtures
so the structural conditions hold. Empirically verified:
- `flat` n=10: fires=0 (htf_bias never sets in 10 bars) → blocks no_htf_bias.
- `killzone_active` n=500: blocks include no_dol/no_htf_bias.
- `sweep_and_fvg` n=500: fires>0 (sweep+FVG sequence fires).

## Module specs

### workspace/backtesting/data_loader.py
- `DataLoader(backend=os.getenv("KYROS_DATA_BACKEND","yfinance"))`
- `load(start, end) -> Path` — returns cached parquet path.
- Cache path: `workspace/data/NQ_1m_{start}_{end}.parquet` (per spec).
- Canonical schema: timestamp(UTC tz-aware), open, high, low, close, volume.
- csv backend: read KYROS_CSV_PATH, parse `date`→timestamp UTC, drop extras
  (contract), filter [start,end], write parquet. No network imports.
- yfinance backend: download /NQ=F in 7-day chunks, stitch, write parquet.
  (Mocked in tests; never runs real HTTP in CI.)
- alpaca backend: requests to Alpaca v2 bars, page, write parquet.
  (Mocked in tests; keys from env, never hardcoded.)
- Cache hit: second load() returns existing parquet without re-reading source.

### workspace/backtesting/calibrator.py
- `CalibrationReport` dataclass: period, total_1m_candles, gate_blocks,
  soft_triggers, fires_by_killzone, fires_by_month, total_fires,
  estimated_llm_cost_usd.
- `TriggerCalibrator(window, builder, trigger, cooldown)` — note: spec
  signature is `(window, builder, trigger, cooldown)`.
- `run(source) -> CalibrationReport` — iterates source, builds snapshots,
  evaluates TriggerEngine in isolation (no LLM). Records every gate block
  and every soft trigger. Calls `cooldown.update()` on fires (D7).
- Writes `workspace/calibration_report.json`.
- `estimated_llm_cost_usd = total_fires * 0.003`.
- gate_blocks keys: no_killzone, no_htf_bias, no_dol, cooldown_active
  (mirrors TriggerEngine gate order — short-circuit, first failure wins).
- soft_triggers keys: fvg, ifvg, sweep, displacement (mirrors TriggerEngine).

### workspace/backtesting/outcome.py
- `TradeOutcome` dataclass: result, candles_to_fill, candles_to_resolution,
  fill_price, exit_price, actual_rr.
- `OutcomeSimulator()` — stateless.
- `simulate(alert, subsequent_candles) -> TradeOutcome`.
- PRECONDITION: caller passes only candles after alert.timestamp. We do NOT
  validate inside simulate (per spec: "No validation of this inside
  simulate() — caller's responsibility"). The test that passes the alert
  candle as first subsequent candle documents why this is wrong.
- no_trade → immediate return, all numeric None.
- Fill: iterate until price range overlaps entry_zone. fill_price = entry_mid.
- Resolution: from candle AFTER fill candle. Long win high>=target, long loss
  low<=stop. Short win low<=target, short loss high>=stop. Both→loss.
- Killzone expiry: compare candle.timestamp vs alert.valid_until. No fill by
  valid_until → no_fill. Filled but unresolved by valid_until → expired.
- actual_rr per D3.

### workspace/backtesting/engine.py
- `TradeTrace` dataclass: trace_id, timestamp, instrument, killzone,
  trigger_reason, snapshot_summary, raw_llm_output, alert, rr_validated,
  outcome.
- `BacktestEngine(loop, simulator, output_path)`.
- `run(source) -> list[TradeTrace]`.
- Resume: read existing output_path, collect processed timestamps, skip.
- For each alert: collect subsequent_candles from replay buffer (after alert),
  call simulator, build TradeTrace, append JSON line.
- Replay buffer: rolling ≥480 1m candles per timeframe.
- snapshot_summary: compact dict (no raw candle arrays).

### workspace/backtesting/report.py
- `PerformanceReport()`.
- `generate(traces) -> str` — writes workspace/backtest_report.md, returns md.
- profit_factor = sum(winning actual_rr) / abs(sum(losing actual_rr)).
- max_drawdown_r: running R cumsum, track peak, max(peak - current).
- expectancy = mean(actual_rr, no_trade/no_fill/expired as 0).
- golden_match_rate: load golden_alerts.json, match traces within 15 min,
  same direction.
- system_prompt_hash: sha256(ICT_SYSTEM_PROMPT)[:8].
